# vicara v2.1.0

v2.1.0 は、Git Worktree を使った日常開発の安定性を高めつつ、停止していたクロスプラットフォーム Release を復旧するためのリリースです。
ローカルリポジトリをアプリ都合で汚さない merge 導線へ改修し、macOS / Linux を含む配布パイプラインを再び有効化しました。

## Highlights

### 🔀 Git Worktree マージの安定化
- `.vicara-worktrees/` の ignore 管理を tracked な `.gitignore` から `.git/info/exclude` へ移行
- 既存プロジェクトに残っている app 由来の `.gitignore` 差分を、安全条件付きで自動移行
- `merge_worktree` 実行前に project root の dirty 状態を preflight check し、分かりやすいエラーで事前停止

### 🧵 Unix PTY の同期境界見直し
- `claude_runner.rs` の `ProcessKiller` 格納制約を `Box<dyn ProcessKiller + Send>` に整理
- `portable-pty` の非 `Sync` 制約で macOS / Linux ビルドが止まる経路を解消
- Windows / Unix の kill / wait / cleanup 経路は維持したまま、所有境界だけを適正化

### 📦 Cross-Platform Release Recovery
- GitHub Actions の Release workflow で macOS Intel / Apple Silicon / Linux の matrix を再有効化
- macOS target 追加 step と Ubuntu の Tauri v2 依存パッケージ導入を明示
- draft release 運用を維持しつつ、全 desktop matrix を継続検証できる構成へ更新
