# vicara v2.2.0

v2.2.0 は、EPIC47 から EPIC54 までをまとめたリリースです。  
レトロスペクティブの保存基盤から UI、AI による振り返り活用、Try からルールへの接続、そして AI 実行基盤そのものの品質改善までを一気通貫で整え、Vicara の「学習して次のスプリントに生かす」ループを実運用レベルまで引き上げました。

## Highlights

### レトロスペクティブ基盤の完成
- レトロ用の DB スキーマを追加し、`retro_sessions` / `retro_items` / `retro_rules` / `project_notes` を永続化できるようになりました。
- レトロ画面を正式実装し、完了済みスプリントの選択、KPT 3 カラム、カード追加・編集・削除・承認、SM サマリ表示まで扱えるようになりました。
- PO アシスタント横に「ふせん」UI を追加し、気づきをその場で残し、Keep / Problem / Try へすぐ転記できるようになりました。
- Story / Task / Sprint にプロジェクト単位の採番を導入し、レトロや履歴画面でも文脈を追いやすくしました。

### AI によるレトロ活用の強化
- Claude CLI のリアルタイムストリーミングを修正し、Dev ターミナルで thinking を読みやすく確認できるようになりました。
- エージェント実行ログを `agent_retro_runs` / `agent_retro_tool_events` として構造化保存し、レトロの材料として再利用できるようになりました。
- 各ロールの振り返りを自動生成し、さらに SM エージェントが KPT を統合してサマリ化できるようになりました。
- PO アシスタントが会話中の気づきをふせんやレトロ項目として追加できるようになり、CLI / API の両経路でレトロ連携が動くようになりました。

### Try から次スプリントの行動へ接続
- 承認済み Try をルールとして ON / OFF できるようになり、次回以降のエージェント実行に反映できるようになりました。
- Settings にレトロルール管理画面を追加し、一覧表示、有効 / 無効切替、編集、削除、手動追加に対応しました。
- エージェントのタスク実行プロンプトへアクティブルールを自動注入し、レトロで決めた改善方針を次の開発へつなげられるようになりました。

### AI 実行基盤とプレビュー運用の品質改善
- 巨大化していた `ai.rs` を責務ごとに分割し、`agent_runner` / `team_leader` / `agent_retro` も再編して保守性を大幅に改善しました。
- 共通層に残っていた `claude_*` 命名を `agent_*` 系へ整理し、Claude 固有実装と共通実装の境界を明確にしました。
- Dev Agent / PO アシスタント / task generation / retro 系のプロンプトを見直し、完了条件・自己検証・出力制約をより明確にしました。
- Preview 起動前の URL 抽出、依存 self-heal、Windows の残留プロセス cleanup、Scaffolding 後の Node 依存 bootstrap を強化し、worktree 運用を安定化しました。
- Preview で見つけた差分をコメント付きで Dev エージェントへ再開発依頼できるようにし、レビューから修正までの往復を短くしました。

## Included Epics

### EPIC47: レトロ DB スキーマ
- レトロセッション、KPT アイテム、レトロルール、プロジェクトノートの保存基盤を追加
- スプリント完了時に draft retro session を自動作成

### EPIC48: レトロ UI
- RetrospectiveView を追加し、KPT 操作と SM サマリ表示に対応
- Story / Task / Sprint の採番表示とプロジェクトラベルを整備

### EPIC49: ふせん UI
- PO アシスタントサイドバーに「ふせん」タブを追加
- ふせんからレトロへ即時転記できる UX を整備

### EPIC50: Claude ストリーミングとレトロログ
- Claude CLI の stream-json によるリアルタイム表示を修正
- エージェント実行ログの構造化保存を導入

### EPIC51: SM エージェント KPT 合成
- 各ロールのレトロレビュー生成と SM による KPT 合成を実装
- レトロ導線まわりのバグ修正と PBI 用語統一を実施

### EPIC52: PO アシスタントのレトロ連携
- PO アシスタントがふせん追加とレトロ項目提案を行えるように拡張
- CLI のマルチアクション実行計画と UI 自動更新を整備

### EPIC53: Try → ルール
- 承認済み Try をルール化して ON / OFF 管理できるように拡張
- ルールをエージェント実行プロンプトへ注入
- Preview URL 動的抽出や Scaffolding 規約整備など周辺運用品質も改善

### EPIC54: AI 品質改善
- AI 関連モジュールの責務分割と generic naming 化を実施
- prompt 品質改善、preview / 依存 self-heal、worktree 安定化を実施
