# vicara v2.0.0

v2.0.0 は、vicara の最も大きなアーキテクチャ変更を含むメジャーリリースです。
Claude Code CLI 専用だった実行基盤を **マルチCLI対応** に抜本的に刷新し、Gemini CLI / Codex CLI を含む3つのコーディングエージェントと、4つのAPIプロバイダー（Anthropic / Gemini / OpenAI / Ollama）をサポートする **ベンダーニュートラルな AI 開発環境** へと進化しました。

## Highlights

### 🔀 マルチCLI実行基盤（CliRunner trait）
- **Claude Code CLI / Gemini CLI / Codex CLI** の3つのコーディングエージェントに対応
- `CliRunner` trait による抽象化で、新しい CLI の追加が容易な設計
- チーム設定でロールごとに使用する CLI を選択可能

### 🌐 マルチAPIプロバイダー対応
- **Anthropic API / Gemini API / OpenAI API / Ollama** の4つの API プロバイダーに対応
- POアシスタント、Inception Deck、タスク分解で使用するプロバイダーを選択可能
- Ollama によるローカル LLM 推論にも対応

### 🛠️ セットアップ状態の可視化
- 設定画面に **セットアップステータスタブ** を追加
- CLI エージェントのインストール状態、API キーの登録状態をひと目で確認可能

### 💬 POアシスタントの進化
- CLI ベースの実行にも対応し、API / CLI の両方で動作可能に
- 使用するプロバイダーの選択が柔軟に

### 🧙 Inception Deck の改善
- ウィザード体験を刷新し、より直感的な対話フローに
- スクラム用語に不慣れなユーザーにも使いやすい表現に改善

### 🎨 設定UIの刷新
- AI設定・チーム設定・セットアップステータスを統合した新しい設定画面
- マルチプロバイダー時代に合わせた設定の整理

### 📦 リリース自動化
- GitHub Actions によるクロスプラットフォームビルド（Windows / macOS Intel / macOS Apple Silicon / Linux）
- タグプッシュによる自動リリース

### 📄 ドキュメントの国際化
- README を英語ベースに再構成し、日本語版も提供
- LLM セットアップガイドを英語・日本語で分離
- Apache License 2.0 を適用

---

## Breaking Changes

> ⚠️ v1.x からの主な非互換変更

- **CLI実行基盤の変更**: 内部のエージェント実行アーキテクチャが `CliRunner` trait ベースに変更されました。既存の設定データは基本的に互換性がありますが、チーム設定で `cli_type` の再設定が必要になる場合があります。
- **API設定の拡張**: AI設定に新しいプロバイダー（OpenAI / Ollama）のフィールドが追加されました。

---

## Internal Improvements

- `CliRunner` trait による CLI 実行の抽象化（Claude / Gemini / Codex）
- `rig` crate を活用した API プロバイダーの統一的な呼び出し
- Windows（`std::process::Command` + パイプ）とUnix（`portable-pty`）でのクロスプラットフォーム対応
- Transport Layer の統一（API / CLI 間の切り替えを透過的に処理）
- Git Worktree の安全機構とプレビュー連携の改善
