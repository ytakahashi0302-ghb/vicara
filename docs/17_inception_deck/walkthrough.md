# Epic 4: AI Inception Deck (Sprint 0) Walkthrough

Epic 4 の「AIインセプションデッキ」機能の実装が完了しました。POの皆様からのフィードバックを反映し、AIとの対話を通じてプロジェクトの方向性を定義し、各種マークダウンファイルを生成・管理する機能を開発しました。

## 実装された機能

### 1. ワークスペースディレクトリ連携
- データベースの `projects` テーブルに `local_path` カラムを追加しました。
- アプリ上部のヘッダーに、[FolderOpen] アイコンを追加しました。
- `@tauri-apps/plugin-dialog` API を利用して、ユーザーがプロジェクトの物理的なローカルディレクトリを選択・保存できるようにしました。

### 2. Live Preview 付きの全画面分割UI
- 従来のカンバン画面から切り替えて使用する、「全画面（フルスクリーン）」の Inception Deck 画面を実装しました。
- **左ペイン (Chat / Wizard)**: AI との対話用UI。現在のフェーズ（1〜5）を表示し、AIと壁打ちを行えます。
- **右ペイン (Live Document / Tabs)**: 生成・編集中のドキュメントをリアルタイムにプレビューするタブUI。
  - `PRODUCT_CONTEXT.md`
  - `ARCHITECTURE.md`
  - `Rule.md`

### 3. Wizard State Management & フェーズ連動タブUI
- 以下の5フェーズの進行を管理するステートマシンを実装しました。
  - Phase 1: Why (プロダクトのコア価値とターゲット)
  - Phase 2: Not List (やらないことリスト)
  - Phase 3: What (技術スタックとアーキテクチャの制約)
  - Phase 4: How (プロジェクト固有の開発ルールや追加ルール)
  - Phase 5: Review (完了)
- AIとの対話進行（Phase）に合わせて、右ペインのアクティブなタブが自動で切り替わるUXを実現しています。

### 4. 既存ファイル読み込みと流用（再利用）
- ディレクトリ設定時および Inception Deck 起動時に、選択されたディレクトリ内に既にあるファイルを読み込みます。
- ファイルが一つでも存在する場合、AIの初期メッセージが自動的に変わり、「既存のファイルが見つかりました。この内容をベースに修正を加えますか？」と問いかけるロジックを実装しました。

### 5. `Rule.md` の先行生成と差分マージ（Append）機能
- Rustバックエンド内に、MicroScrum AI標準の「ベースルール」を定数として定義しました。
- ファイルが存在しない状態でディレクトリ設定が完了すると、自動的にベースルールが `Rule.md` に物理出力をされます。
- Phase 4（How）の対話終了時、POがAIと合意した「プロジェクト固有の追加ルール」のみを、既存の `Rule.md` の末尾にテキストベースで追記（マージ）する仕様を実現しました。

### 6. RAG統合（プロンプト生成の改善）
- 既存のAIタスク生成および機能提案のRAG検索（`ai.rs` 内 `build_project_context`）を拡張し、指定したディレクトリ下の `PRODUCT_CONTEXT.md`、`ARCHITECTURE.md`、`Rule.md` を読み取り、システムプロンプトに結合するように機能強化しました。
- Inception Deck自体のシステムプロンプトにも、ダラダラと終わらない一問一答を防ぐための強い制御（質問を1〜2回に限定し自らネクストアクションを促す）を組み込んでいます。

## 検証方法（POによるマニュアルテスト手順）

現在、ローカルでのビルド（`cargo check` / `npm run lint`等）を通過している状態です。以下の手順で動作をご確認ください。

1. **ディレクトリの選択**
   - アプリを起動し、ヘッダーの「フォルダ未設定」ボタンをクリックして、任意のテスト用ローカルフォルダを選択してください。
2. **Inception Deckへの遷移とPhase 1の開始**
   - ヘッダーの「Inception Deck」ボタンをクリックします。全画面UIになり、Phase 1が開始されることを確認してください。
3. **AIとの対話**
   - 左ペインでAIと対話し、適切に内容が深掘りされること、さらにAI側からフェーズの移行を提案されるか確認してください。
4. **タブ連動とファイル生成**
   - Phaseが進むごとにタブが切り替わり、合意に至った段階で右ペインのプレビューが更新され、指定したローカルディレクトリに各 `*.md` ファイルが生成されることを確認してください。
   - 特に Phase 4終了時、ベースルールに加えて追記内容が `Rule.md` に正しく Append されているかをチェックしてください。
5. **既存ファイルの流用**
   - ファイルを生成したのちに、別のプロジェクトを作成し、同じディレクトリを設定してみてください。
   - Inception Deck起動時に「既存のファイルが見つかりました」とAIが認識し、ファイル内容が引き継がれるか確認してください。
