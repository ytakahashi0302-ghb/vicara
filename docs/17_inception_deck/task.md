# 17_inception_deck: タスクリスト

## 1. ワークスペースディレクトリ設定と初期化
- [x] `projects`テーブルに`local_path`（TEXT）カラムを追加するDBマイグレーションを作成・実行する。
- [x] プロジェクト設定（またはヘッダー）に、`@tauri-apps/plugin-dialog` APIを用いたローカルディレクトリ選択UIを追加する。
- [x] Rust側にプロジェクトの `local_path` を更新するTauriコマンド実装し、同時に、選択されたディレクトリ内に既存の3ファイル（`PRODUCT_CONTEXT.md`, `ARCHITECTURE.md`, `Rule.md`）が存在するかチェックし、存在状況をフロントに返す。

## 2. Inception Deck 全画面分割UIの実装
- [x] カンバン画面から遷移できる、Inception Deck用の「全画面（フルスクリーン）」ルーティング・コンポーネントを作成する。
- [x] 画面を左右に分割するレイアウトを実装する。
  - 左ペイン: AIとのチャットインターフェース（メッセージ履歴、入力フォーム）
  - 右ペイン（タブUI）: ライブドキュメントプレビュー。タブ構成は `PRODUCT_CONTEXT.md`, `ARCHITECTURE.md`, `Rule.md`。手動での切り替えを可能にする。

## 3. スプリント0 ウィザード状態管理と既存ファイルの流用
- [x] `useState` や `useReducer` を用いて、Phase 1〜5の状態管理（ステートマシン）を実装する。
- [x] **フェーズ連動タブ切り替え**: 左ペインのPhase進行に合わせて右ペインのアクティブタブを自動的に切り替える処理を実装する。
  - Phase 1 & 2 (Why, Not List): `PRODUCT_CONTEXT.md` 
  - Phase 3 (What): `ARCHITECTURE.md`
  - Phase 4 (How): `Rule.md`
- [x] **既存ファイルの流用処理**:
  - 各Phaseの開始時、該当ファイルが存在する場合は内容を右ペインに表示する。
  - AIの初期メッセージとして「既存ファイルの内容をベースに修正するか、次へ進むか」を問いかけるロジックを組み込む。

## 4. `Rule.md` の先行生成と差分マージ（Phase 4）
- [x] アプリ側（Rustまたは定数）にMicroScrum AIの「ベースルール」を定義する。
- [x] プロジェクトディレクトリ設定時、またはPhase 4開始時に、`Rule.md` が未存在ならベースルールを書き出して物理的に作成（既存ならスキップまたはマージ準備）するRustコマンドを実装する。
- [x] Phase 4にて右ペインにベースルールを表示し、「プロジェクト固有の追加ルールはありますか？」とAIにヒアリングさせる。ヒアリング後、Rust側でファイルに追記（マージ）する。

## 5. RAG統合（ファイル内容のプロンプト結合）
- [x] 既存のRAGコンテキスト生成処理（`ai.rs`）を拡張し、`local_path`上の3ファイルを読み込んでシステムプロンプトに結合する処理を実装。

## 6. プロンプト制御最適化
- [x] 各フェーズ用System Promptに「1〜2回の深掘り後、現状まとめを提示し、次へ進むか必ず確認する」強い制約を含める。
