# Epic56: スプリント中のルートディレクトリ動作確認ボタン

## 目的
- アクティブスプリント画面から、現在選択中プロジェクトのルートディレクトリを対象に動作確認を起動できるようにする。
- 既存の Review タスク向け worktree プレビューとは責務を分け、スプリント進行中の簡易確認導線を追加する。

## タスク
- [x] スプリントボードのヘッダーに「動作確認」ボタンを追加する
- [x] ルートディレクトリ向けのプレビュー判定ロジックを共通化する
- [x] project root を対象にした Tauri preview コマンドを追加する
- [x] root preview の再表示・停止導線を追加する
- [x] task マージ成功時に root preview を自動停止する
- [x] 既存の TaskCard プレビューが回帰しないことを確認する
- [x] ビルド・テストで動作確認する

## 受け入れ条件
- [x] アクティブスプリント画面に「動作確認」ボタンが表示される
- [x] `local_path` 未設定時は安全にエラー案内される
- [x] React/Vite 等のプロジェクトではルートディレクトリから dev サーバーを起動できる
- [x] 静的サイトではルートの `index.html` を直接開ける
- [x] 起動中の root preview を UI から再表示・停止できる
- [x] task のマージ成功後、古い root preview が自動停止される
- [x] 既存の Review タスクのプレビュー機能が維持される
