# Epic 21 引き継ぎ書 (Handoff to Next Developer / Agent)

次世代自律エージェント基盤に向けたフロントエンドの大改修（3ペインUI実装）が完了しました。
今後の機能開発やリファクタリングにおいて、以下の内容を遵守してください。

## 1. 実装完了サマリー
- **3ペインレイアウト**: メイン画面を「左ペイン(70%)」と「右ペイン(30%)」に分割。さらに左ペインを「Top(Kanban)」と「Bottom(Terminal Dock)」のフレックスで構築。
- **UI/UXデザイン**: TailwindCSS v4の機能を活かし、Glassmorphism(`backdrop-blur`)、影(`shadow`)による立体感とマイクロアニメーション（Hover, 回転等）を適用。
- **独立したスクロールと状態**: Kanban部分は`flex-1`および`overflow-hidden`で自身の内部のみスクロール可能。右ペインのPO Agent Sidebarは非表示時アンマウントではなく`w-0`で隠すアプローチを取ったため、チャット履歴のStateは保持されます。

## 2. 【警告】技術的な重要ポイント (罠の回避録)
本環境においてReact Strict ModeやDOM描画タイミングに関連する致命的なバグを解消済みです。
**以下のロジックは不用意にリファクタリング（簡略化）しないでください！**

> [!CAUTION]
> **Strict ModeにおけるPTYセッションのライフサイクル管理**
> `src/hooks/usePtySession.ts` では、Reactの「マウント → アンマウント → マウント」の順で発火する二重呼び出しに対応するため、`setTimeout` (50ms) による遅延実行と、クリーンアップ関数内での確実な `pty_kill` を実装しています。これを単純な `useEffect` でマウント直後に `pty_spawn` するように書き換えると、バックエンドにゾンビプロセスが残り、ターミナルの出力が二重になるバグが再発します。

> [!WARNING]
> **xterm.js と ResizeObserver の描画タイミング制御**
> `src/components/terminal/TerminalDock.tsx` での `fitAddon.fit()` 呼び出しには、厳密なガードを施しています。
> パネルが「最小化状態」等でマウントされたとき、`clientWidth` や `clientHeight` が 0 の状態で `fit()` を呼ぶと xterm 内部で例外（dimensions undefined）が投げられ画面操作が止まります。
> 修正版では `offsetWidth > 0` などの厳密なサイズ確認に加え、`requestAnimationFrame` でブラウザのDOMペイント完了を待ってから実行し、さらに `try-catch` でラップしています。この非同期制御とガード節は不可欠です。
