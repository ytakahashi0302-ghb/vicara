# Epic 21: 次世代自律エージェント向け 3ペインUIの実装 計画

## 1. 目的と概要
Epic 20で完成したRust(Tauri)の「Rig AI基盤」と「PTYターミナル基盤」を活用し、Reactフロントエンドに「POエージェント（俯瞰・統制）」「進行管理（カンバン）」「Devエージェント（実行・監視）」の3ペインUIを構築する。

## 2. ユーザーレビュー要否 (User Review Required)
> [!IMPORTANT]
> 以下の点について、実装に入る前にPO（ユーザー）の確認と承認をお願いします。
> 1. **ペインのリサイズ機能**: 初回実装では設計書の指示通り「割合指定の固定幅（Left:Right = 70:30, Top:Bottom = 60:40）」で実装して良いか、あるいは境界線をドラッグして幅/高さを可変（Resize）にすべきか？
> 2. **ターミナル入力**: PTYでのターミナル（xterm.js）は、当面はAIの実行したコマンド出力を「監視」するRead-Onlyの目的がメインか、それともユーザーが直接手動でコマンドを入力して操作できるようにすべきか？

## 3. UIアーキテクチャ設計
- **全体レイアウト (Zoning)**
  - 画面全体を横断的に `flex` または `grid` を使って分割。
  - **Left Pane (70%)**: タスク管理および実行状況の監視エリア。
    - **Top (60%) / Kanban**: 既存のカンバンボードを表示。高さを制限し、内部でのみスクロール可能にする。
    - **Bottom (40%) / Terminal Dock**: `xterm.js` をマウント。Devエージェントのログターミナルエリア。
  - **Right Pane (30%)**: POエージェント等と会話するためのサイドバーエリア。

## 4. 実装アプローチ

### Phase 1: パッケージ導入と3ペインレイアウトの基盤構築
- `xterm`, `xterm-addon-fit` などを導入。
- アプリケーションのメイン画面 (`App.tsx` あるいは `Layout.tsx`) を3ペインレイアウトにリファクタリング。
- 既存の Kanban UI コンポーネントを Left-Top 領域に埋め込み、はみ出さない（Overfowスクロールが内部で閉じる）よう Tailwind CSS を整理。

### Phase 2: Terminal Dock と PTY 機能の統合
- TauriのPTY API (`pty_spawn`, `pty_execute`, `pty_kill`) を呼び出しやすくするカスタムフック (`usePtySession.ts` 等) の実装。
- Reactコンポーネント (`TerminalDock.tsx`) に `xterm.js` を初期化・バインド。
- コンポーネントのマウント/アンマウントに合わせてセッションの開始・破棄を確実に行うクリーンアップ処理の実装。
- ウィンドウのリサイズに応じて `xterm-addon-fit` がターミナル表示サイズを最適化する仕組みを導入。

### Phase 3: PO Agent Sidebar の実装 (Rig 統合)
- コンポーネント (`PoAgentSidebar.tsx`) の作成。
- バックエンドの `chat_with_team_leader` API を呼び出し、リアルタイムでのチャットUI（会話履歴の表示・送信）を実装。
- ローディングやエラーなどのステータス表示の追加。

### Phase 4: UI/UXの最適化とテスト
- **美観 (Aesthetics)**: プロジェクトルールの「Premium Designs」「動的なデザイン」に基づき、モダンで洗練されたGlassmorphism、適度なアニメーション、美しいカラーパレットを配したデザインに各ペインをブラッシュアップ。
- **結合テスト**: PTYのコマンド実行・出力テスト。Windowsの出力文字コードの不整合（CP932）や改行の表示崩れへの対応を検証。

## 5. テスト・検証計画 (Verification Plan)
### 手動検証
1. **レイアウト**: 3ペインの割合が仕様（70:30, 60:40）通りになっており、各ペインの内部スクロールが独立して動作するか。
2. **ターミナル**: ターミナルへマニュアルテスト用コマンドを送信し、出力結果が適切に xterm.js に描画されるか。コンポーネントアンマウント時にプロセスが安全に `pty_kill` されるか。
3. **チャット連携**: 右ペインにてRig基盤を使用したボットと会話が可能か。
