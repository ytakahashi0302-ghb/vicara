# Epic 39: 設定画面 - セットアップ状況タブ タスクリスト

## ステータス

- 状態: `Ready`
- 着手条件: Epic 36 完了（CLI 検出コマンドが利用可能であること）
- 作成日: 2026-04-09

## 概要

GlobalSettingsModal に「セットアップ状況」タブを新設し、環境の準備状態（Git、各 CLI、API キー）を一覧で確認できるようにする。未セットアップ項目がある場合はこのタブをデフォルト表示する。

## 実行順序

### 1. SetupStatusTab コンポーネントの作成
- [ ] `src/components/ui/SetupStatusTab.tsx` を新規作成する。
- [ ] 以下の情報をカード形式で表示する:
  - **環境ツール**
    - Git: インストール状態 + バージョン
    - Claude Code CLI: インストール状態 + バージョン
    - Gemini CLI: インストール状態 + バージョン
    - Codex CLI: インストール状態 + バージョン
  - **API キー**
    - Anthropic: 設定済み / 未設定
    - Gemini: 設定済み / 未設定
- [ ] 各項目にステータスアイコン（チェック / 警告）を表示する。
- [ ] 未インストールの CLI には簡潔なインストール手順リンクを添える。
- [ ] 「再検出」ボタンを配置し、CLI 検出を手動でリフレッシュできるようにする。

### 2. API キー設定状態の検出
- [ ] Tauri store (`settings.json`) から API キーの有無を取得するロジックを実装する。
- [ ] キーの値自体は表示せず、「設定済み」「未設定」のみ表示する。

### 3. GlobalSettingsModal へのタブ追加
- [ ] `src/components/ui/GlobalSettingsModal.tsx` のタブ構成を変更する:
  - 新タブ順: セットアップ状況 / POアシスタント設定 / チーム設定 / プロジェクト設定
- [ ] 未セットアップ項目がある場合（CLI が1つもインストールされていない、または API キーが未設定）、セットアップ状況タブをデフォルトで開くロジックを追加する。

### 4. 最低要件の明示
- [ ] タブ内に以下のガイダンスを表示する:
  - 「Dev エージェント機能には、最低1つの CLI ツール + Git が必要です」
  - 「PO アシスタント機能には、API キー設定 または CLI ツールが必要です」（Phase 3 以降で更新）

### 5. 動作確認
- [ ] 各ツールのインストール有無が正しく表示されること。
- [ ] API キーの設定有無が正しく表示されること。
- [ ] 再検出ボタンで CLI 検出結果が更新されること。
- [ ] 未セットアップ時にタブがデフォルトで選択されること。
