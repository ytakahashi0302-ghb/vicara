# Epic 40: 設定画面 - チーム設定 CLI 種別対応 タスクリスト

## ステータス

- 状態: `Ready`
- 着手条件: Epic 37 完了（DB に cli_type カラムが存在すること）、Epic 38 完了（全 Runner が利用可能であること）
- 作成日: 2026-04-09

## 概要

チーム設定タブ（TeamSettingsTab）をリニューアルし、各ロールに CLI 種別（Claude / Gemini / Codex）を選択できるドロップダウンを追加する。CLI 種別に応じてモデル選択肢を動的に切り替える。

## 実行順序

### 1. TeamSettingsTab に CLI 種別セレクタを追加
- [ ] `src/components/ui/TeamSettingsTab.tsx` の各ロールカードに CLI 種別ドロップダウンを追加する。
- [ ] 選択肢: `Claude Code` / `Gemini CLI` / `Codex CLI`
- [ ] 選択値を `TeamRoleSetting.cli_type` にバインドする。
- [ ] 未インストールの CLI には `(未検出)` ラベルを付与し、選択はできるが警告を表示する。

### 2. モデル選択の CLI 連動
- [ ] CLI 種別の変更時にモデル選択肢をリセットする。
- [ ] 各 CLI のデフォルトモデル:
  - Claude: `claude-sonnet-4-20250514`
  - Gemini: `gemini-2.5-pro`
  - Codex: `o3`
- [ ] CLI 種別変更時にデフォルトモデルを自動設定する。
- [ ] カスタムモデル入力（テキスト直接入力）は引き続きサポートする。

### 3. CLI 検出結果との連携
- [ ] `useCliDetection()` フック（Epic 36）からインストール状態を取得する。
- [ ] インストール済みの CLI にはバージョン情報をサブテキストで表示する。
- [ ] 未インストール CLI を選択したロールが存在する場合、保存ボタン付近に警告メッセージを表示する。

### 4. 保存処理の更新
- [ ] `save_team_configuration` の呼び出しに `cli_type` を含める。
- [ ] 既存ロールの CLI 種別が正しく保存・読み込みされることを確認する。

### 5. シードデータの更新
- [ ] `db.rs` の初期ロール（seed-lead-engineer）に `cli_type = 'claude'` を明示的に設定する。

### 6. 動作確認
- [ ] ロールの CLI 種別を変更して保存 → 再読み込みで値が維持されること。
- [ ] CLI 種別変更時にモデルがデフォルト値にリセットされること。
- [ ] 未インストール CLI 選択時に警告が表示されること。
- [ ] 保存後にタスク実行で正しい CLI が使用されること。
