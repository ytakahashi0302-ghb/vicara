# Epic 24: ClaudeCLIを用いたタスクの自律実行基盤の構築 (最終実装仕様)

本ドキュメントでは、カンバン上のタスクを起点に Claude CLI (Claude Code) を外部プロセスとして起動し、リアルタイムにストリーミング出力と状態管理を行う基盤の最終的な実装詳細を記述します。

## 1. プロセス実行戦略 (OS対応)

Windows環境における仮想ターミナル (ConPTY) の制約を回避するため、プラットフォームごとに異なる実行戦略を採用しています。

*   **Windows**: `std::process::Command` を使用し、`Stdio::piped()` で標準出力・標準エラー出力を直接キャプチャします。これにより、ConPTY特有の出力バッファリングやハングの問題を完全に回避しています。
*   **macOS / Linux**: `portable-pty` を使用し、リッチなエスケープシーケンスやANSIカラー出力を最大限に活かせる仮想ターミナル (PTY) モードで実行します。

## 2. コマンド構築とプロンプト供給

改行や日本語、特殊文字を含む複雑なプロンプトを確実に Claude CLI に渡すため、コマンド引数 (`-p`) ではなく **標準入力 (stdin) 経由** でプロンプトを供給する設計に変更されました。

*   **基本コマンド**: `claude --permission-mode bypassPermissions --add-dir <cwd> --output-format stream-json --verbose`
*   **プロンプト供給**: プロセス起動直後に stdin へプロンプト文字列を書き込み、EOT (Ctrl-D) を送信することで Claude に処理を開始させます。これにより、OSのコマンドライン引数制限やクォート崩壊のリスクを排除しています。

## 3. プロセス管理と生存監視

プラットフォームごとの差異を吸収するため、`ProcessKiller` trait を導入し、セッション管理を抽象化しています。

*   **ClaudeSession**: 実行中の子プロセス、および stdout/stderr 読み取り用のハンドルを保持し、Tauri の State 内で管理することで、関数の終了に伴う不意のプロセス終了 (Drop) を防ぎます。
*   **ストリーミング**: 専用の読み取りスレッド (Windows は stdout/stderr それぞれ) を起動し、Tauri イベント `claude_cli_output` を通じてフロントエンドへリアルタイムにログを配信します。
*   **ガードレール**: 180秒の強制タイムアウトスレッドを配置し、API呼び出しの暴走によるコスト増大を防止します。

## 4. フロントエンド連携

*   **TerminalDock**: `xterm.js` を利用。Tauri イベントをリッスンし、受信したテキストを逐次描画します。
*   **ステータス遷移**: プロセス開始時に `In Progress` へ遷移し、完了時に自動的に `Done` へステータスを更新します。
