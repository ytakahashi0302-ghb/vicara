# Epic 34 Handoff: vicara ブランド継承メモ

## 最重要事項

本プロダクトの正式名称は **vicara** である。  
今後の Epic・実装・ドキュメント・UI 文言・設定値において、旧名称 `MicroScrum AI` / `ai-scrum-tool` を復活させてはならない。

## すでに変更済みの基盤識別子

- Tauri app identifier: `com.vicara.app`
- npm package 名: `vicara`
- Rust crate 名: `vicara`
- Rust lib 名: `vicara_lib`
- SQLite DB 名: `vicara.db`
- worktree ディレクトリ: `.vicara-worktrees`
- localStorage key: `vicara.*`

将来の Epic では、これらの識別子を前提として扱うこと。  
旧名称との互換を取るための戻し実装は、技術的負債を再導入するため避ける。

## プロダクト思想の継承

`README.md` に記載した通り、vicara は「AI に勝手に進めさせるツール」ではない。  
人間がプロダクトオーナーとして **何を作るか、どの順序で進むか、何を採用するか** を意思決定し、AI はその判断に従って実装・調査・検証を加速する。

今後の実装では、必ず以下を守ること。

- 人間中心の意思決定を UI と体験の中心に置く
- AI の自律性よりも、透明性・レビュー可能性・制御可能性を優先する
- ソロ開発者が複数の AI を率いる「チーム体験」を損なわない
- スクラムを、人間と AI の共通言語として活用する

## ドキュメント運用の注意

- ルートの公式ドキュメントは `vicara` 基準で維持する
- `docs/` 配下の過去 Epic 履歴は、履歴保全のため旧名称を含んだまま残している
- 過去履歴を一括置換しないこと

## 次の担当 AI へのメッセージ

Epic 34 により、ブランド刷新は完了している。  
今後の Epic では、名称の揺れを増やさず、README に記載された **人間の意思決定を中心に据える開発環境** という思想を最優先で継承してほしい。  
判断に迷った場合は、「この変更は vicara の人間中心思想を強めるか、それとも AI に主導権を渡しすぎるか」を基準に設計判断を行うこと。
