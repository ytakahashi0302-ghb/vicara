# EPIC45 引き継ぎメモ

## このEpicで確立したこと
- 設定画面は、従来の肥大化した単一モーダルから、洗練されたマスターディテール UI へ進化した。
- 左側で対象セクションを選び、右側で詳細を編集する構成に整理されたことで、画面幅を問わず直感的に操作しやすい情報設計になっている。
- 各セクションは `src/components/ui/settings/` 配下の共通コンポーネント群で統一され、今後の拡張でも見た目と実装の一貫性を保ちやすい。

## クイックスイッチャー
- POアシスタント画面と Inception Deck 画面には、AI プロバイダーとモデルを即座に切り替えられるクイックスイッチャーが実装されている。
- 切り替え結果は設定ストアと連動し、裏側の AI 利用設定へ即時反映される前提で設計されている。
- AI 利用先の選択は実行画面側で行い、設定画面の AI プロバイダー設定は API キー / 接続情報の管理に責務を絞っている。

## UI / デザインシステム
- 設定 UI 全体で視覚的ノイズを極限まで削ぎ落とし、過剰な色分けや不要な装飾を整理した。
- エッジタブハンドル、設定カード、ロール一覧などもニュートラルなトーンへ揃え、Vicara 全体のデザインシステムが確立されつつある。
- POアシスタント設定やチーム設定は、情報の意味ごとに階層化された構造へ再設計済みで、今後の追加要件も載せやすい。

## 次のEpic担当AIへの注意点
- `src/context/**`, `src/hooks/**`, `src/types/**` は既存資産の中核として極力触らない方針が維持されている。
- 設定関連の新規拡張は、まず `SettingsShell`, `SettingsSidebar`, `SettingsSection`, `SettingsField`, `SettingsContext` の責務に沿って実装すること。
- チーム設定や POアシスタント設定は、UI 密度と視線誘導を重視して最終調整済みのため、再編集時は「機能追加」だけでなく「面のバランス」を必ず確認すること。
