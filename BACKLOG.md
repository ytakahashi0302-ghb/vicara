## UI / UX & ルーティング関連
- [ ] **Inception Deckへの再入線UI欠如**: カンバン画面等から「インセプションデッキを再度見直す・修正する」ための明確なUI動線（ルーティング）が存在しない。

- [ ] **プロジェクト削除機能のUI欠如**: バックエンドに `delete_project` コマンドはあるが、フロントエンドから呼び出して削除するUIが未実装。
- [ ] **AIタスク生成ボタンの消失**: Epic 6の基盤改修（Backlog分離）の際、「ストーリーからAIが子タスクを自動生成する（generate_tasks_from_story）」ボタンがバックログ画面(`BacklogView.tsx`)へ再配置されずに消失した。復旧・配置見直しが必要。

## AI連携 & LLM挙動の差異
- [ ] **AIモデルバージョンのハードコード**: Rust側 (`ai.rs`) でGemini/Anthropicのモデル名が決め打ち。設定画面（Settings）から利用モデルを選択できる仕組みを導入する。
- [ ] **ClaudeのChattyな応答によるJSONパースエラー**: インセプションデッキ作成等で、Claude（Sonnet等）が自然言語の前置きを含めてしまいJSONパースに失敗する。システムプロンプトの厳格化、または Tool use (Function calling) の活用で堅牢な仕組みへ改修する。
- [ ] **LLMの特性差異によるドキュメント全上書きバグ**: インセプションデッキ生成プロセスで、Claudeを使用すると既存のファイル内容（Phase 1等）が新しいセクションで全上書き・消失してしまう。プロンプトで「追記・マージ」を厳格に指示するか、バックエンドの書き込みロジックを差分更新型に改修する。
- [ ] **AIタスク生成時のJSONパース正規表現の脆弱性**: `generate_tasks_from_story` 内のJSON抽出処理が最短マッチの正規表現 (`(?s)\[.*?\]`) で行われており、ネストされた配列などでパース失敗の可能性がある。将来的に Structured Output (`application/json`) の利用を検討。
- [ ] **AI呼び出しコスト(予算・トークン)の未計測**: コスト管理実現のため、AI呼び出しごとの usage(input/output tokens) を記録し、チャット単位でリアルタイムにコスト使用量を画面上で確認できるUIを実装する。

## アーキテクチャ & 状態管理・安全性
- [ ] **タイマー稼働中の新規プロジェクト作成ガード漏れ**: スプリントタイマーが稼働中（RUNNING）にヘッダーから新規プロジェクトを作成すると、Interaction Guardが発動せず状態が引き継がれてしまう。レアケースとして現状許容中。
- [ ] **WorkspaceContext の初期プロジェクトIDハードコード**: `currentProjectId` の初期値が `'default'` と固定されている。削除された場合のフォールバック（最初の有効なプロジェクトを選択する等）が未実装。
- [ ] **CRUD操作時の全件再フェッチ**: ストーリー等の追加・更新時に現状は `fetchStories()` 等でリストを全件再取得している。データ量増加時の遅延・チラつき防止のため、楽観的UI (Optimistic UI) への段階的移行を検討する。
- [ ] **TauriコマンドのContext未公開**: `update_project` や `delete_project` が WorkspaceContext 経由で呼び出せるようになっていないため、呼び出し経路が統一されていない。
- [ ] **未使用コード・デバッグログの残留**: Tauriテンプレート由来の `greet` コマンドや、`db.rs` 内の `println!` が残っている。リリース前にクリーンアップする。
