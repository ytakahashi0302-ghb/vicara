# MicroScrum AI 🤖

> **AIと一緒にスクラムを回す、ソロ開発者のための開発パートナー**

アイディアをメモするだけで、AIがUser Storyに整理し、タスクに分解し、スプリントを管理してくれます。
スプリントを重ねるたびに、あなた専用のAIチームへと育っていきます。

---

## なぜ「スクラム」なのか

スクラムは人間が長年かけて磨いてきた開発プロセスです。
このツールでは、**スクラムを人間とAIの共通言語**として使います。

- **人間側**：スクラムを知っていれば、AIが何をしているか分かる → 安心して任せられる
- **AI側**：「次はプランニング」「次はレトロ」と状態遷移が明確 → 曖昧な指示でも動ける
- **お互い**：レトロスペクティブで振り返りができる → スプリントごとにAIチームが改善されていく

## 開発フロー

```
┌──────────────────────────────────────────────────────┐
│  スプリントプランニング（＋バックログリファインメント）       │
│                                                      │
│  人間：アイディアメモ・壁打ち                            │
│  AI ：User Story化 → 粒度チェック → タスク分解          │
│       優先度整理 → スプリントゴール設定                   │
└──────────────────────┬───────────────────────────────┘
                       ↓
┌──────────────────────────────────────────────────────┐
│  スプリント実行（1〜8時間）                              │
│                                                      │
│  人間：カンバン確認、方向性の判断                         │
│  AI ：タスク自律実行（サブエージェント）                   │
└──────────────────────┬───────────────────────────────┘
                       ↓
┌──────────────────────────────────────────────────────┐
│  レビュー ＋ レトロスペクティブ                           │
│                                                      │
│  人間：成果物確認・フィードバック                         │
│  AI ：各エージェントが担当を振り返り → SQLiteに蓄積        │
│       次スプリントへの改善提案を自動生成                   │
└──────────────────────┬───────────────────────────────┘
                       ↓
                 （次のループへ）
```

**人間の介入ポイントはプランニング・実行中の判断・レビューの3回。**
イベントをシンプルに絞ることで、ソロ開発者が無理なく回せるリズムを実現します。

> 参考：[AIエージェントだけでスクラムを回してみた（Qiita, 2026）](https://qiita.com/shyamagu/items/0ff827a1c2deef68ba9a)
> 5体のAIによるフル自律スクラムの実験。本ツールはその「人間が主役として関わり続ける」バージョンを目指しています。

## 主な機能

### 現在実装済み

| 機能 | 説明 |
|------|------|
| 🤖 **AIタスク分解** | User StoryからタスクをAIが自動生成（Anthropic API / Gemini API） |
| 📋 **インタラクティブカンバン** | Story（親）とTask（子）の紐付けを可視化。ドラッグ＆ドロップ操作 |
| ⏱️ **マイクロスクラム管理** | 1スプリント＝1〜8時間。専用タイマーとステータス追跡 |
| 📝 **Markdownサポート** | タスク詳細のリッチプレビュー |
| 📊 **スプリント履歴** | 完了スプリントのアーカイブ、ベロシティ確認 |
| 🔒 **完全ローカル保存** | 全データをSQLiteに保存。外部クラウドへのデータ流出なし |
| 🤝 **タスク自律実行** | Claude CLI (Claude Code) と連携し、タスクを自律的に遂行（ファイル編集・検証） |

### 開発予定

| 機能 | 説明 |
|------|------|
| 💬 **壁打ちエージェント** | アイディアメモからUser Storyへ会話形式で整理。プランニングとリファインメントを同時に実現 |
| 🔄 **エージェントレトロ** | 各エージェントが担当を振り返り、改善内容をSQLiteに蓄積。次スプリントに自動反映 |

## AIチームが育つ仕組み

スプリントを重ねるごとに、各エージェントが自分の仕事を振り返ります。

```
タスク分解エージェント  →「粒度が大きすぎたタスクはなかったか」
実行エージェント       →「詰まったポイントはどこか、次回どう改善するか」
```

振り返り内容はSQLiteに蓄積され、次回プランニング時に自動で参照されます。
使えば使うほど、あなたの開発スタイルに合ったAIチームになっていきます。

## 類似ツールとの違い

| | Devin / Claude Code | MetaGPT | **MicroScrum AI** |
|---|---|---|---|
| 人間の関与 | 最初と最後のみ | ほぼなし | **壁打ち・レトロで継続関与** |
| プロセスの透明性 | ブラックボックス | ブラックボックス | **スクラムで可視化** |
| 継続的改善 | なし | なし | **レトロで毎スプリント改善** |
| データ管理 | クラウド | クラウド | **完全ローカル** |
| 対象 | チーム / 全自動 | 全自動 | **ソロ開発者、人間がオーナー** |

## セットアップ

### 必要環境

- [Rust](https://www.rust-lang.org/)
- [Node.js](https://nodejs.org/)
- Anthropic API キー または Gemini API キー
- **Claude Code CLI** (Anthropic)
  - `npm install -g @anthropic-ai/claude-code` でインストールし、`claude login` を済ませておく必要があります。

### 起動手順

```bash
# 依存関係のインストール
npm install

# 開発モードで起動
npm run tauri dev
```

### 開発コマンド

```bash
npm run dev          # Vite開発サーバー起動
npm run build        # フロントエンドビルド
npm run lint         # ESLint + 型チェック
npm run tauri dev    # Tauriアプリ開発モード起動
npm run tauri build  # 本番パッケージ作成
```

## 開発スタイル：このツール自体がコンセプトの実証

MicroScrum AIは、AIエージェントを使って開発されています。

- **Gemini 3.1 Pro（ブラウザ）**：エージェントへの指示プロンプトを設計・生成
- **Antigravity × Gemini 3.1 Pro High**：コード生成・ファイル操作を実行

「AIに何をさせるかを考えるAI」と「実際に手を動かすAI」を分けるこの構造は、
このツールが目指す人間とAIの役割分担そのものでもあります。
